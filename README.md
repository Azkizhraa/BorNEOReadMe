## Non-Tech Track Explanation

This project participates in the **Non-Technical Track**, focusing on user-centered design and solution conceptualization rather than full system implementation. The development process follows a **Design Thinking approach**, emphasizing empathy, user research, and iterative prototyping.

Through demographic analysis and persona creation, the team identified key challenges faced by citizens in disaster-prone areas, such as misinformation, delayed hazard reporting, and limited access to clear evacuation guidance. These insights guided the design of the SiagAI prototype.

The result is a **Figma-based prototype** that demonstrates the user experience and interaction flow of the platform. Core features include a personalized disaster dashboard, a misinformation-checking chatbot, and a community reporting feed. While the technical architecture is conceptually described, the primary deliverable in this track is the **design, usability, and problem–solution framework** of the system.
